# Part2-3장. 포인터란 무엇인가

## 1. 포인터란

- 포인터: 주소를 저장하는 변수

  - 메모리의 주소를 저장하고 있다가 해당 주소로 데이터를 전달, 참조
  - 메모리 주소를 참조해서 다양한 자료형 변수에 접근, 조작
  - 배열과 같은 연속된 데이터에 접근, 조작
  - 동적 메모리 영역(힙영역)에 접근, 조작

  → 두 변수의 값을 바꾸는 함수의 매개변수로 사용

- 주소는 상수, 포인터는 변수  
  → 포인터 값을 바꾸거나, 여러 개의 포인터에 같은 값을 넣을 수 있음
- 주소와 포인터의 크기는 같음 → 4byte

## 2. 포인터 변수의 선언과 사용

- 포인터 변수 선언

  - `(자료형)* (포인터 변수 이름)=NULL;` (NULL 포인터 설정)
    - 자료형의 `*`을 이용해서 값을 참조할 수 있다는 뜻
  - 포인터 변수를 null(0이라는 뜻)로 초기화하는 것이 시스템에 안정적
    ex) `int* pointer=NULL;`
  - 모든 포인터 변수의 크기는 4바이트임
  - 가리키는 변수 형이 같으면 연속 선언 가능
    ex) `int* pa, * pb;`
    ```c
    char c='A';
    char* cp=&c;
    ```
    - `cp == &c`
      `*cp == *&c == c`
      `(간접 접근) (직접 접근)`

- 포인터 변수 초기화

  ```c
  int num=10;
  int* ip=NULL;
  ip=&num;

  int num=10;
  int* ip=&num;
  ```

  - 주소를 구할 변수를 먼저 선언했다면 선언과 동시에 초기화하는 것 가능

- 잘못 사용된 포인터
  - 포인터 변수에 주소를 저장하지 않음
  - 포인터 변수에 이상한 주소 저장
  - 포인터는 가리키는 자료형이 같을 때만 대입
    - 할당되지 않은 영역을 다른 변수가 이미 쓰고 있으면 변수 값이 바뀌기 때문
      (형변환 사용 → 경고 메시지 없지만 대입 후 사용에 문제없어야 함)
      > `int* p=NULL;` → `p`가 변수 (`*p`는 `p`가 가진 주소의 값을 참고하는 것)
- `const` 포인터: 가리키는 변수의 값 바꿀 수 없음 (포인터값 고정 X)
  - 다른 주소 대입 가능 (포인터값 고정 X)
  - 변수 자체 값은 변경할 수 있지만 간접접근으로 변경할 수 없음
  - 대부분 문자열 상수에 사용
